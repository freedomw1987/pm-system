📌 1. 項目概述

項目內容
系統名稱Internal PM System（內部項目管理系統）
用戶規模約 50 人
部署環境公司內部伺服器（Docker）
認證方式Email + Password（管理員創建帳號）
技術架構React + Tailwind (前端) / Elysia.js + Bun (後端) / PostgreSQL

👥 2. 用戶角色

角色主要職責
項目經理/產品經理管理項目及需求
開發技術主管根據需求拆解任務，安排開發人員
開發人員處理任務，更新狀態，登記工作時數
測試人員提交缺陷，修復後驗證，登記工作時數
管理員帳號管理、權限管理、報表查看
💡 重要特性：
  • 一個用戶可以有多個角色，不同項目中角色可能不同
  • 項目經理只能看到自己負責的項目
  • 開發人員只能看到被指派的任務

⚙️ 3. 核心功能模組

3.1 項目管理

  • 創建/編輯/刪除項目
  • 設置項目成員及角色
  • 項目狀態追蹤（active / completed / archived）

3.2 需求管理

  • 創建/編輯/刪除需求（可附帶附件）
  • 需求狀態：pending / in_progress / completed

3.3 任務管理

  • 技術主管根據需求拆解任務
  • 支援跨需求任務（一任務可關聯多個需求）
  • 任務可附帶附件
  • 指派給開發人員
  • 狀態：pending / in_progress / testing / completed

3.4 缺陷管理

  • 測試人員提交缺陷，關聯到具體任務
  • 缺陷狀態：open / in_progress / resolved / closed
  • 嚴重程度：low / medium / high / critical

3.5 工作時數

  • 每次任務完成後或每天登記（可補填過往記錄）
  • 精確到小時，保留兩位小數（如：0.25 = 15分鐘）
  • 可關聯任務或缺陷

3.6 報表功能

報表類型使用者內容
項目成本報表管理員、PM各人員工作時數統計
項目進度報表管理員、PM需求/任務完成進度百分比

📊 6. 數據模型

核心實體：

  • User - 用戶（email, name, passwordHash）
  • Project - 項目（name, description, status）
  • ProjectMember - 項目成員（project_id, user_id, role）- 每項目獨立角色
  • Requirement - 需求（project_id, title, status）
  • Task - 任務（title, assignee_id, status, estimatedHours）
  • TaskRequirement - 任務-需求關聯（多對多，支援跨需求）
  • Bug - 缺陷（task_id, title, status, severity）
  • WorkLog - 工作時數（user_id, task_id/bug_id, hours, workDate）
  • Attachment - 附件（entity_type, entity_id, filename, storedPath）
  • RefreshToken - Refresh Token（token, userId, expiresAt）

🚀 7. 開發階段

Phase 1：基礎架構（1-2 週）

  • ✅項目初始化（React + Elysia.js + PostgreSQL）
  • ✅Docker Compose 編排
  • ✅數據庫遷移（Migrations）
  • ✅用戶認證（JWT）
  • ✅用戶管理（Admin）

交付：可登入的系統骨架

Phase 2：核心功能（2-3 週）

  • ✅項目 CRUD
  • ✅項目成員管理（按項目角色）
  • ✅需求 CRUD（含附件上傳）
  • ✅任務 CRUD（含跨需求關聯）
  • ✅缺陷 CRUD

交付：項目/需求/任務模組

Phase 3：工作流程（2 週）

  • ✅工作時數登記（支持補填）
  • ✅任務狀態流轉
  • ✅缺陷狀態流轉
  • ✅權限控制（各角色）

交付：缺陷+時數+權限

Phase 4：報表與優化（1-2 週）

  • ✅項目成本報表
  • ✅項目進度報表
  • ✅RWD 響應式優化
  • ✅測試與部署

交付：完整 MVP

⏱️ 總計：6-9 週

🌐 8. 部署架構

  • Nginx - HTTPS reverse proxy（self-signed 測試） + SPA fallback
  • Frontend - React SPA，Port 3000
  • Backend - Elysia.js API，Port 4000
  • PostgreSQL - Port 5432，數據持久化
  • Uploads - 文件存儲，Docker volume 持久化
